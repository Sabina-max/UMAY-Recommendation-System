## 1. Итоговые результаты и выводы

В ходе практики был разработан и протестирован базовый прототип рекомендательной системы (Rule-Based Recommendation System) для маркетплейса UMAY Shop на основе доступных публичных данных.

### Ключевые выводы:
1. **Очистка данных:** При анализе выгрузки из API (`/recommend/product/list`) обнаружилось, что около трети записей являлись дубликатами (повторяющиеся ID или одинаковые названия товаров от разных поставщиков). После удаления дубликатов был сформирован очищенный рабочий набор данных.
2. **Аномалии в полях:** В исходных данных системное поле `score` оказалось незаполненным. В качестве рейтинга было решено использовать поле `showScore`, так как оно содержало реальные оценки пользователей.
3. **Ограниченность выборки:** Так как публичный API выдает уже отобранные популярные товары, их базовые метрики (продажи и оценки) почти одинаковы. Обычная сортировка по ним не работает, поэтому потребовалось внедрение весовых коэффициентов и нормализации.
4. **Фильтрация контента:** Текстовый фильтр по ключевым словам позволил успешно разделить взрослый и детский ассортимент в сценарии карточки товара, чтобы детские вещи не рекомендовались во взрослых категориях.

## 2. Формула ранжирования (Scoring)

Финальный балл товара ($FinalScore$) рассчитывается как сумма его нормализованных метрик, умноженных на веса под конкретный экран:

$$FinalScore = w_{sales} \cdot Sales_{norm} + w_{score} \cdot Score_{norm} + w_{discount} \cdot Discount_{norm} + w_{promo} \cdot Promo + w_{sim} \cdot Similarity$$

* Метрики продаж, рейтинга и скидки нормализованы в диапазоне от 0 до 1 внутри текущих границ датафрейма, чтобы их можно было корректно складывать.
* $Promo$ — равен 1, если товар участвует в промо-акции, и 0, если нет.
* $Similarity$ — индекс сходства, который используется только на странице товара (включает совпадение подкатегории и сходство цен по функции Гаусса в пределах $\pm40\%$).

### Экспертные веса по сценариям:

| Сценарий | Продажи ($w_{sales}$) | Рейтинг ($w_{score}$) | Скидка ($w_{discount}$) | Промо ($w_{promo}$) | Сходство ($w_{sim}$) |
| :--- | :---: | :---: | :---: | :---: | :---: |
| **1. Главная страница** | **0.35** | 0.30 | 0.20 | 0.15 | 0.00 |
| **2. Карточка товара** | 0.05 | 0.15 | 0.10 | 0.10 | **0.60** |
| **3. Пустая корзина** | 0.25 | 0.20 | **0.35** | 0.20 | 0.00 |

*Весовые коэффициенты были выбраны экспертным способом в соответствии с задачами каждого пользовательского сценария.*

## 3. Примеры рекомендаций для 3 сценариев (Top-5 выдача)

### 📋 Сценарий 1: Главная страница
*Стратегия: Разнообразие витрины. Категории чередуются, чтобы пользователь видел разный ассортимент.*
1. **Детские нескользящие шлепанцы** (Детская обувь) — *Высокий рейтинг, стабильный спрос.*
2. **Маленький альбом большого объема** (Канцелярия) — *Высокий рейтинг, есть скидка.*
3. **Шелковая майка на бретельках** (Женская одежда) — *Стабильный спрос, высокий рейтинг.*
4. **Ручная роспись чайного сервиза** (Посуда) — *Высокий рейтинг и стабильный спрос.*
5. **Мужские летние свободные шорты** (Мужская одежда) — *Участник промо-акции.*

### 👕 Сценарий 2: Карточка товара (Пример: Женская водолазка)
*Стратегия: Контекстные аналоги. В выдаче только женская одежда похожей цены, детский контент полностью скрыт.*
1. **Модная женская футболка весна/осень** — *Совпадение категории, близкая цена.*
2. **Долгий рукав верхний слой для женщин** — *Высокий рейтинг, близкая цена.*
3. **Свободная футболка с V-вырезом** — *Совпадение подкатегории, действует скидка.*
4. **Черная короткая рубашка для женщин** — *Близкая цена, высокий рейтинг.*
5. **Футболка из хлопка в полоску** — *Совпадение категории, хорошее качество.*

### 🛍️ Сценарий 3: Пустая корзина
*Стратегия: Финансовые триггеры. Максимальный упор на товары с хорошими скидками для мотивации к покупке.*
1. **Маленький альбом большого объема** — *Популярный товар со скидкой 12.5%.*
2. **Детские нескользящие шлепанцы** — *Стабильный спрос, выгодная цена.*
3. **Мужские рабочие шорты** — *Высокая скидка, участник промо.*
4. **Очищающий скраб-гель для душа** — *Действующая скидка, высокий рейтинг.*
5. **Женская короткая футболка с рельефом** — *Хорошая скидка, стабильный спрос.*

## 4. Описание архитектуры для Production-версии

При переносе системы в реальную рабочую среду (Production) и при наличии полной базы данных, рекомендуется перейти на **двухуровневую схему рекомендаций**:

1. **Этап 1 (Retrieval / Отбор кандидатов):** Быстрый отбор 200–300 наиболее подходящих товаров из всего каталога маркетплейса на основе категории текущего товара или базовых интересов пользователя. Для ускорения поиска можно использовать кэширование в БД **Redis**.
2. **Этап 2 (Ranking / Финальное ранжирование):** Точечная сортировка отобранных кандидатов с помощью библиотек машинного обучения (например, **CatBoost**). На этом этапе модель учитывает сотни признаков: историю кликов, просмотров, покупок и добавлений в корзину.
3. **Автоматизация пайплайнов:** Сбор данных, очистка и регулярное обновление весов и моделей настраиваются через планировщик задач **Apache Airflow**.

## 5. Рекомендации для бизнеса UMAY Shop

1. **Техническое улучшение данных:** Навести порядок в API: устранить дубликаты карточек и восстановить работу поля `score` на бэкенде.
2. **Объяснение рекомендаций пользователю:** Добавить краткое объяснение рекомендации («Высокий рейтинг», «Часто покупают», «Товар со скидкой»), чтобы повысить прозрачность алгоритма для покупателя.
3. **Проведение A/B тестов:** Запустить тест и сравнить текущую выдачу маркетплейса с алгоритмом чередования категорий. Целевая метрика — увеличение кликов по рекомендациям и рост добавлений в корзину.
4. **Сбор истории действий:** Начать собирать подробные логи поведения пользователей (клики, добавления в избранное, время просмотра). Без этих данных невозможно построить точную персональную рекомендательную систему.
